Agent Reference

Think of agents as your expert teammates who never sleep. You've got 30 specialists here--each one obsessed with a single domain. Seven of them are WordPress-specific reviewers covering PHP standards, Gutenberg blocks, hooks, themes, JavaScript, frontend races, and test quality. Plus an AI Building Blocks reviewer for the Abilities API. Call them individually when you need focused expertise, or orchestrate them together for multi-angle analysis.

How to Use Agents

# Basic invocation
claude agent [agent-name]

# With a specific message
claude agent [agent-name] "Your message here"

# Examples
claude agent wp-php-reviewer
claude agent security-sentinel "Audit the payment flow"
claude agent wp-gutenberg-reviewer "Review the pricing table block"

Review Agents (19)

Your code review dream team. Seven WordPress-specific reviewers plus eleven framework-agnostic specialists that catch what humans miss--security holes, performance cliffs, architectural drift, hook priority conflicts, and missing block deprecations. They're picky. They disagree with each other. That's the point.

wp-php-reviewer

WordPress

Your WordPress PHP expert with a high quality bar. Reviews code for WordPress coding standards, security patterns, and maintainability. Whether you're building a plugin settings page or a custom REST API endpoint, this reviewer enforces proper nonce verification, capability checks, and data sanitization.

Key Focus Areas

  • WordPress PHP Coding Standards compliance
  • Security patterns (nonces, capabilities, sanitization, escaping)
  • REST API endpoint conventions
  • Settings API and Options API usage
  • Proper use of WordPress functions over raw PHP
claude agent wp-php-reviewer "Review the settings page implementation"

wp-javascript-reviewer

WordPress

Reviews JavaScript code for WordPress block editor, Interactivity API, and @wordpress/* package patterns. Knows the difference between wp.element and React, catches improper block.json usage, and ensures your Interactivity API stores handle state correctly.

Key Focus Areas

  • Block editor JavaScript patterns
  • Interactivity API store and directive patterns
  • @wordpress/* package usage
  • block.json completeness and correctness
  • Proper script enqueuing and dependencies
claude agent wp-javascript-reviewer "Review the testimonial block code"

wp-hooks-reviewer

WordPress

The WordPress hook system is powerful but fragile. This agent reviews your actions, filters, and custom hooks for correct timing, priority conflicts, and architectural patterns. It catches the bugs that happen when two plugins register at the same priority or when you hook into init when you should hook into wp_loaded.

Key Focus Areas

  • Correct hook timing (init vs wp_loaded vs plugins_loaded)
  • Priority conflict detection
  • Hook removal matching (matching add_action/remove_action signatures)
  • Custom hook naming conventions
  • Filter return value consistency
claude agent wp-hooks-reviewer "Review the notification system hooks"

wp-gutenberg-reviewer

WordPress

Custom Gutenberg blocks need specialized review beyond what a general JS reviewer catches. This agent checks deprecation handling (did you add a deprecation entry when changing the save function?), block.json schema completeness, server-side rendering patterns, and InnerBlocks usage.

Key Focus Areas

  • Deprecation handling for save function changes
  • block.json schema and attribute completeness
  • Server-side rendering (render.php) patterns
  • InnerBlocks and nested block patterns
  • Block API version compatibility
claude agent wp-gutenberg-reviewer "Review the pricing table block"

wp-theme-reviewer

WordPress

Block themes and classic themes have very different requirements. This agent reviews theme.json compliance, template hierarchy correctness, Full Site Editing patterns, and proper asset enqueuing. Whether you're building a new block theme or converting a classic theme to FSE, it catches the structural issues.

Key Focus Areas

  • theme.json schema compliance and settings
  • Template hierarchy and template parts
  • Full Site Editing (FSE) patterns
  • Proper asset enqueuing (wp_enqueue_style/script)
  • Classic-to-block theme migration
claude agent wp-theme-reviewer "Review the block theme structure"

wp-frontend-races-reviewer

WordPress

Race conditions in WordPress JavaScript are subtle and dangerous. This agent reviews Interactivity API stores for concurrent async operations, block editor async save handlers, AJAX/REST request ordering, and DOM event timing. If your infinite scroll loads duplicate posts or your filter dropdown shows stale results, this reviewer will tell you why.

Key Focus Areas

  • Interactivity API store async race conditions
  • Block editor concurrent save operations
  • AJAX/REST API request ordering and cancellation
  • Stale response handling
  • DOM event timing and cleanup
claude agent wp-frontend-races-reviewer "Review the infinite scroll implementation"

wp-ai-building-blocks-reviewer

WordPress

Reviews code that uses WordPress AI Building Blocks--the Abilities API, AI Client SDK, and MCP Adapter. Checks correct hook timing for ability registration, schema validation, permission callbacks, provider-agnostic patterns, and output sanitization. Essential when building AI-powered WordPress features.

Key Focus Areas

  • Abilities API hook timing and schema validation
  • AI Client SDK provider-agnostic patterns
  • MCP Adapter configuration
  • Permission callback presence
  • Output sanitization for AI-generated content
claude agent wp-ai-building-blocks-reviewer "Review the AI excerpt generation"

wp-test-reviewer

WordPress

Reviews test suites for WordPress plugins and themes. Checks test isolation, meaningful assertions, security path coverage (nonce-fail tests, capability checks, malicious input), integration patterns with real WordPress (WP_UnitTestCase), naming conventions, and common anti-patterns like mocking core when integration tests are available.

Key Focus Areas

  • Test isolation -- no shared state or execution order dependency
  • Meaningful assertions -- no empty tests, prefer assertSame over assertEquals
  • Security path coverage -- nonce failures, capability denials, malicious input
  • Integration coverage -- hooks, DB operations, REST endpoints with real WordPress
  • Naming conventions -- test methods describe scenarios
  • Anti-patterns -- mocking core unnecessarily, testing implementation details
claude agent wp-test-reviewer "Review the tests for the REST API endpoint"

security-sentinel

Security

Security vulnerabilities hide in boring code--the "just grab the user ID from params" line that ships a privilege escalation bug to production. This agent thinks like an attacker: SQL injection, XSS, auth bypass, leaked secrets. Run it before touching authentication, payments, or anything with PII.

Security Checks

  • Input validation analysis
  • SQL injection risk assessment
  • XSS vulnerability detection
  • Authentication/authorization audit
  • Sensitive data exposure scanning
  • OWASP Top 10 compliance
  • Hardcoded secrets search
claude agent security-sentinel "Audit the payment flow"

performance-oracle

Performance

Your code works fine with 10 users. What happens at 10,000? This agent time-travels to your future scaling problems--unoptimized queries that murder your database, O(n^2) algorithms hiding in loops, missing indexes, memory leaks. It thinks in Big O notation and asks uncomfortable questions about what breaks first when traffic spikes.

Analysis Areas

  • Algorithmic complexity (Big O notation)
  • Query optimization and index usage
  • Memory management review
  • Caching opportunity identification
  • Network usage optimization
  • Frontend bundle impact
claude agent performance-oracle

architecture-strategist

Architecture

Every "small change" either reinforces your architecture or starts eroding it. This agent zooms out to see if your fix actually fits the system's design--or if you're bolting duct tape onto a crumbling foundation. It speaks SOLID principles, component boundaries, and API contracts.

Analysis Areas

  • Overall system structure understanding
  • Change context within architecture
  • Architectural violation identification
  • SOLID principles compliance
  • Component boundary assessment
  • API contract evaluation
claude agent architecture-strategist

data-integrity-guardian

Data

Database changes can't be easily rolled back once they're in production. This agent is your last line of defense before you accidentally drop data, create a race condition in transactions, or violate data privacy requirements. It obsesses over referential integrity, rollback safety, and data constraints.

Review Areas

  • Migration safety and reversibility
  • Data constraint validation
  • Transaction boundary review
  • Referential integrity preservation
  • Privacy compliance (GDPR, CCPA)
  • Data corruption scenario checking
claude agent data-integrity-guardian

pattern-recognition-specialist

Patterns

Patterns tell stories--Factory, Observer, God Object, Copy-Paste Programming. This agent reads your code like an archaeologist reading artifacts. It spots good patterns (intentional design), anti-patterns (accumulated tech debt), dead code, and duplicated blocks you swore you'd refactor later. Includes dead code detection for identifying unused functions and unreachable paths.

Detection Areas

  • Design patterns (Factory, Singleton, Observer, etc.)
  • Anti-patterns and code smells
  • Dead code detection and unreachable paths
  • TODO/FIXME comments
  • God objects and circular dependencies
  • Naming consistency and code duplication
claude agent pattern-recognition-specialist

code-simplicity-reviewer

Quality

Simplicity is violent discipline. This agent asks "do you actually need this?" about every line, every abstraction, every dependency. YAGNI isn't a suggestion--it's the law. Your 200-line feature with three layers of indirection? This agent will show you the 50-line version that does the same thing.

Simplification Checks

  • Analyze every line for necessity
  • Simplify complex logic
  • Remove redundancy and duplication
  • Challenge abstractions
  • Optimize for readability
  • Eliminate premature generalization
claude agent code-simplicity-reviewer

agent-native-reviewer

Agent

Reviews code to ensure agent-native parity--any action a user can take through the UI, an agent should also be able to take through APIs or tools. When you add a new feature, this agent checks whether it's accessible to automated agents, not just human users clicking buttons.

Key Focus Areas

  • UI feature accessibility via API/tools
  • Agent-equivalent workflows for user actions
  • System prompt and tool design review
  • Automation gap identification
claude agent agent-native-reviewer "Review the email filtering feature"

call-chain-verifier

Traceability

Traces UI-initiated actions through all WordPress layers and verifies signatures at each boundary. From an AJAX form through the wp_ajax_ handler, through service logic, to the database write--this agent maps every step and checks for broken connections, missing permission checks, and unvalidated data at layer boundaries.

Verification Areas

  • AJAX form to handler chain tracing
  • REST API route to database chain
  • Block editor save to storage chain
  • Permission callback verification at each boundary
  • Data validation between layers
claude agent call-chain-verifier "Trace the AJAX form save chain"

schema-drift-detector

Database

Detects unrelated database schema changes in WordPress PRs by cross-referencing dbDelta calls, table creation, and version tracking. Schema drift in WordPress happens when activation hooks run locally with different plugin versions. This agent catches schema changes that don't belong in the current PR.

Detection Areas

  • dbDelta call cross-referencing
  • Unrelated table creation or modification
  • Version tracking consistency
  • Activation hook schema changes
claude agent schema-drift-detector "Review the database changes in this PR"

data-migration-expert

Data

Validates data migrations, backfills, and production data transformations against reality. When PRs involve ID mappings, column renames, enum conversions, or schema changes, this agent verifies the mappings match production data and checks for swapped values and rollback safety.

Validation Areas

  • ID mapping verification
  • Enum conversion accuracy
  • Backfill completeness
  • Rollback procedure safety
  • Production data compatibility
claude agent data-migration-expert "Review the status enum migration"

deployment-verification-agent

Deploy

Produces Go/No-Go deployment checklists with SQL verification queries, rollback procedures, and monitoring plans. When PRs touch production data, migrations, or risky data changes, this agent creates concrete pre-deploy and post-deploy verification steps.

Checklist Areas

  • Pre-deployment verification queries
  • Post-deployment validation steps
  • Rollback procedures
  • Monitoring plan setup
  • Go/No-Go decision criteria
claude agent deployment-verification-agent "Create deployment checklist"

Research Agents (5)

Stop guessing. These agents dig through documentation, GitHub repos, git history, and real-world examples to give you answers backed by evidence. They read faster than you, remember more than you, and synthesize patterns you'd miss.

framework-docs-researcher

Research

Official docs are scattered. GitHub examples are inconsistent. Deprecations hide in changelogs. This agent pulls it all together--docs, source code, version constraints, real-world examples. Ask "how do I use the Interactivity API?" and get back patterns that actually work in production.

Capabilities

  • Fetch official framework and library documentation
  • Identify version-specific constraints and deprecations
  • Search GitHub for real-world usage examples
  • Analyze library source code
  • Synthesize findings with practical examples
claude agent framework-docs-researcher "Research Interactivity API patterns"

best-practices-researcher

Research

"Best practices" are everywhere and contradictory. This agent cuts through the noise by evaluating sources (official docs, trusted blogs, real GitHub repos), checking recency, and synthesizing actionable guidance. You get code templates, patterns that scale, and answers you can trust.

Capabilities

  • Leverage multiple sources (Context7 MCP, web search, GitHub)
  • Evaluate information quality and recency
  • Synthesize into actionable guidance
  • Provide code examples and templates
  • Research issue templates and community engagement
claude agent best-practices-researcher "Find WordPress block patterns"

git-history-analyzer

Git

Your codebase has a history--decisions, patterns, mistakes. This agent does archaeology with git tools: file evolution, blame analysis, contributor expertise mapping. Ask "why does this code exist?" and get the commit that explains it.

Analysis Techniques

  • Trace file evolution using git log --follow
  • Determine code origins using git blame -w -C -C -C
  • Identify patterns from commit history
  • Map key contributors and expertise areas
  • Extract historical patterns of issues and fixes
claude agent git-history-analyzer "Analyze changes to the settings module"

repo-research-analyst

Research

Every repo has conventions--some documented, most tribal knowledge. This agent reads ARCHITECTURE.md, issue templates, PR patterns, and actual code to reverse-engineer the standards. Perfect for joining a new project or ensuring your PR matches the team's implicit style.

Analysis Areas

  • Architecture and documentation files (ARCHITECTURE.md, README.md, CLAUDE.md)
  • GitHub issues for patterns and conventions
  • Issue/PR templates and guidelines
  • Implementation patterns using ast-grep or rg
  • Project-specific conventions
claude agent repo-research-analyst

learnings-researcher

Knowledge

Searches docs/solutions/ for relevant past solutions by frontmatter metadata. Before implementing features or fixing problems, this agent surfaces institutional knowledge to prevent repeated mistakes. It finds solutions you've already documented so you don't re-solve the same problem.

Capabilities

  • Search docs/solutions/ by module, symptom, and tags
  • Surface relevant past solutions before coding
  • Cross-reference related documented issues
  • Identify recurring problem patterns
claude agent learnings-researcher "Search for REST API authentication issues"

Design Agents (3)

Design is iteration. These agents take screenshots, compare them to Figma, make targeted improvements, and repeat. They fix spacing, alignment, colors, typography--the visual details that compound into polish.

design-iterator

Design

Design doesn't happen in one pass. This agent runs a loop: screenshot the UI, analyze what's off (spacing, colors, alignment), implement 3-5 targeted fixes, repeat. Run it for 10 iterations and watch rough interfaces transform into polished designs.

Process

  • Take focused screenshots of target elements
  • Analyze current state and identify 3-5 improvements
  • Implement targeted CSS/design changes
  • Document changes made
  • Repeat for specified iterations (default 10)
claude agent design-iterator

figma-design-sync

Figma

Designers hand you a Figma file. You build it. Then: "the spacing is wrong, the font is off, the colors don't match." This agent compares your implementation to the Figma spec, identifies every visual discrepancy, and fixes them automatically.

Workflow

  • Extract design specifications from Figma
  • Capture implementation screenshots
  • Conduct systematic visual comparison
  • Make precise code changes to fix discrepancies
  • Verify implementation matches design
claude agent figma-design-sync

design-implementation-reviewer

Review

Before you ship UI changes, run this agent. It compares your implementation against Figma at a pixel level--layouts, typography, colors, spacing, responsive behavior. Catches the "close enough" mistakes that users notice but you don't.

Comparison Areas

  • Layouts and structure
  • Typography (fonts, sizes, weights)
  • Colors and themes
  • Spacing and alignment
  • Different viewport sizes
claude agent design-implementation-reviewer

Workflow Agents (3)

Tedious work you hate doing. These agents handle the grind--reproducing bugs, resolving PR comments, analyzing specs. They're fast, they don't complain, and they free you up to solve interesting problems.

bug-reproduction-validator

Bugs

Half of bug reports aren't bugs--they're user errors, environment issues, or misunderstood features. This agent systematically reproduces the reported behavior, classifies what it finds, and assesses severity. Saves you from chasing ghosts.

Classification Types

  • Confirmed - Bug reproduced successfully
  • Cannot Reproduce - Unable to reproduce
  • Not a Bug - Expected behavior
  • Environmental - Environment-specific issue
  • Data - Data-related issue
  • User Error - User misunderstanding
claude agent bug-reproduction-validator

pr-comment-resolver

PR

Code review comments pile up. This agent reads them, plans fixes, implements changes, and reports back what it did. It doesn't argue with reviewers or skip hard feedback--it just resolves the work systematically.

Workflow

  • Analyze code review comments
  • Plan the resolution before implementation
  • Implement requested modifications
  • Verify resolution doesn't break functionality
  • Provide clear resolution reports
claude agent pr-comment-resolver

spec-flow-analyzer

Testing

Specs always have gaps--edge cases nobody thought about, ambiguous requirements, missing error states. This agent maps all possible user flows, identifies what's unclear or missing, and generates the questions you need to ask stakeholders.

Analysis Areas

  • Map all possible user flows and permutations
  • Identify gaps, ambiguities, and missing specifications
  • Consider different user types, roles, permissions
  • Analyze error states and edge cases
  • Generate critical questions requiring clarification
claude agent spec-flow-analyzer
