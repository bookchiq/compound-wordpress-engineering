Changelog

All notable changes to the compound-wordpress-engineering plugin. This project follows Semantic Versioning and Keep a Changelog conventions. This is a WordPress fork of EveryInc/compound-engineering-plugin.

v1.5.0

2026-02-27 Testing & Static Analysis

Added

  • wp-playground skill - Start and manage WordPress Playground instances for local testing. Includes startup/stop/status scripts, blueprint patterns reference, and evaluation procedure for comparing Playground implementations.
  • wp-testing skill - Scaffold and run WordPress test suites (PHPUnit, wp-browser, Playwright). Includes scaffold and run scripts, plus 4 reference docs: PHPUnit setup, test patterns, TDD workflow, and fixture patterns.
  • wp-phpcs skill - Run PHP_CodeSniffer with WordPress Coding Standards. Includes setup and run scripts, plus phpcs.xml.dist configuration templates.
  • wp-phpstan skill - Run PHPStan static analysis with WordPress extensions (szepeviktor/phpstan-wordpress). Includes setup and run scripts, plus phpstan.neon configuration templates.
  • wp-eslint skill - Run ESLint with @wordpress/eslint-plugin. Includes run script and .eslintrc.json templates.
  • wp-test-reviewer agent - Reviews test suites for isolation, meaningful assertions, security path coverage, integration patterns, naming conventions, and anti-patterns.
  • Playwright MCP server - Browser automation for end-to-end testing, replaces agent-browser as the primary tool.

Changed

  • wp-php-reviewer agent - Added Step 0: runs PHPCS and PHPStan (if available) before prompt-based analysis, incorporating tool findings into review.
  • wp-javascript-reviewer agent - Added Step 0: runs ESLint (if available) before prompt-based analysis, incorporating tool findings into review.
  • workflows:work command - Added "Establish Testing Strategy" step, red/green TDD checkpoints, test coverage row, and Playwright MCP screenshots.
  • workflows:review command - Added Static Analysis Phase, Test Coverage Check, and wp-test-reviewer as conditional agent.
  • test-browser command - Rewritten to use Playwright MCP as primary tool with agent-browser as fallback. Auto-detects server URL.
  • setup skill - Added test environment detection, static analysis tool detection, new configuration questions, and extended output format.

Fixed

  • Shell script security - Added PORT validation, array-based command construction, PROJECT_NAME sanitization, PID validation before kill, and null-terminated xargs in all static analysis run scripts.
  • Duplicate static analysis - Review agents now skip tool execution when results are already provided by workflows:review.
  • Playwright MCP version - Pinned to @playwright/mcp@0.0.68 (was @latest).
  • scaffold-tests.sh - --type parameter now produces differentiated scaffolding for plugin/theme/block.

v1.0.0

2026-02-21 Initial WordPress Release

WordPress fork of EveryInc/compound-engineering-plugin v2.34.0. Adapted for WordPress theme, plugin, and block development with specialized WordPress agents and skills.

Added

6 WordPress-Specific Review Agents

  • wp-php-reviewer - WordPress PHP coding standards and security patterns
  • wp-javascript-reviewer - Block editor, Interactivity API, and @wordpress/* patterns
  • wp-hooks-reviewer - Hook timing, priority conflicts, and architectural patterns
  • wp-gutenberg-reviewer - Block deprecation handling, block.json, server-side rendering
  • wp-theme-reviewer - theme.json compliance, template hierarchy, FSE patterns
  • wp-frontend-races-reviewer - Race conditions in Interactivity API and AJAX handlers

WordPress Development Skill

  • wp-development-patterns - WordPress coding standards, hook patterns, and best practices

Removed

8 Upstream Agents (not applicable to WordPress)

  • kieran-rails-reviewer - Rails code review
  • dhh-rails-reviewer - DHH-style Rails review
  • kieran-python-reviewer - Python code review
  • kieran-typescript-reviewer - TypeScript code review
  • julik-frontend-races-reviewer - Replaced by wp-frontend-races-reviewer
  • ankane-readme-writer - Ruby gem documentation
  • lint - Ruby/ERB linting
  • every-style-editor - Every Inc. style guide

4 Upstream Skills (not applicable to WordPress)

  • dhh-rails-style - 37signals Rails conventions
  • andrew-kane-gem-writer - Ruby gem patterns
  • dspy-ruby - Ruby LLM applications
  • every-style-editor - Every Inc. style guide

Changed

  • Renamed plugin from compound-engineering to compound-wordpress-engineering
  • Updated all workflow commands to use WordPress-appropriate agents
  • Synced framework-agnostic improvements from upstream v2.34.0
