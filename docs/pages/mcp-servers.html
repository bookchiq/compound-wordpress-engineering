MCP Servers Reference

MCP servers are power tools that plug into Claude Code. The plugin bundles two: Context7 for real-time framework documentation, and Playwright for browser automation and end-to-end testing. Together they let Claude look up how an API works and then test that it does.

Known Issue: Auto-Loading

Sometimes MCP servers don't wake up automatically. If Claude can't look up docs, you'll need to add the server manually. See Manual Configuration for the fix.

Context7

Ever ask Claude about a framework and get an answer from outdated training data? Context7 fixes that. It's a documentation service that keeps Claude current with 100+ frameworks--WordPress, PHP, React, Next.js, whatever you're using. Think of it as having the official docs piped directly into Claude's brain.

Tools Provided

Tool Description
resolve-library-id Maps "WordPress" to the actual library identifier Context7 uses. You don't call this--Claude does it automatically
get-library-docs Fetches the actual documentation pages. Ask "How does the Interactivity API work?" and this grabs the latest WordPress docs

Example Usage

# Just ask about the framework--Claude fetches current docs automatically

"Look up the WordPress Interactivity API documentation"

"How does the Block Editor useBlockProps hook work?"

"What are the best practices for WordPress REST API endpoints?"

# You get answers based on the latest docs, not Claude's training cutoff

Configuration

{
  "context7": {
    "type": "http",
    "url": "https://mcp.context7.com/mcp"
  }
}

Playwright

Browser automation for end-to-end testing. Playwright MCP gives Claude the ability to navigate pages, click elements, fill forms, take screenshots, and verify what users actually see. It's the primary tool for the /test-browser command and integrates with WP Playground for testing WordPress admin interfaces.

Tools Provided

Tool Description
browser_navigate Navigate to a URL
browser_snapshot Capture the accessibility tree of the current page
browser_click Click an element on the page
browser_type Type text into an input field
browser_screenshot Take a screenshot of the current page
browser_wait Wait for a specified time or condition

Example Usage

# Navigate to WordPress admin
mcp__playwright__browser_navigate(url="http://localhost:9400/wp-admin/")

# Take a screenshot of the dashboard
mcp__playwright__browser_screenshot()

# Click an element
mcp__playwright__browser_click(element="Posts")

Configuration

{
  "playwright": {
    "command": "npx",
    "args": ["@playwright/mcp@0.0.68"]
  }
}

Fallback: agent-browser

If Playwright MCP isn't available (e.g., Node.js not installed), the /test-browser command falls back to the agent-browser skill, which uses Vercel's agent-browser CLI for browser automation.

Manual Configuration

If the server doesn't load automatically (you'll know because Claude can't fetch docs), you need to wire it up yourself. It's a one-minute copy-paste job.

Project-Level Configuration

To enable for just this project, add this to .claude/settings.json in your project root:

{
  "mcpServers": {
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    },
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@0.0.68"]
    }
  }
}

Global Configuration

Or enable everywhere--every project on your machine gets these servers. Add to ~/.claude/settings.json:

{
  "mcpServers": {
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    },
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@0.0.68"]
    }
  }
}

Requirements

Server Requirement
Context7 Internet connection (HTTP endpoint, no local install needed)
Playwright Node.js 18+ and npx (installs @playwright/mcp automatically)

Verifying MCP Servers

After you add the config, restart Claude Code. Then test that everything works:

# Ask Claude what it has
"What MCP tools do you have access to?"

# Test Context7 (should fetch real docs)
"Look up WordPress Block Editor documentation"

# If it fails, double-check your JSON syntax and file paths
