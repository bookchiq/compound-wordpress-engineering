Skill Reference

Think of skills as reference manuals that Claude Code can read mid-conversation. When you're building WordPress plugins and want proper hook patterns, or creating Gutenberg blocks with correct deprecation handling, you don't need to paste documentation--just invoke the skill. Claude reads it, absorbs the patterns, and writes code that way.

How to Use Skills

# In your prompt, reference the skill
skill: [skill-name]

# Examples
skill: gemini-imagegen
skill: wp-development-patterns
skill: create-agent-skills

Skills vs Agents

Agents are personas--they do things. Skills are knowledge--they teach Claude how to do things. Use claude agent [name] when you want someone to review your code. Use skill: [name] when you want to write code in a particular style yourself.

WordPress Development (3)

WordPress-specific skills that teach Claude proper coding standards, hook patterns, block development, and AI feature integration for WordPress.

wp-development-patterns

WordPress

WordPress has its own way of doing things--hooks instead of events, the Settings API instead of raw forms, proper nonce verification for security. This skill teaches Claude WordPress coding standards, hook patterns, and development best practices so your plugin and theme code follows the conventions that the WordPress ecosystem expects.

Key Patterns

  • Hook System - Actions, filters, custom hooks, priority management
  • Security - Nonces, capability checks, data sanitization, output escaping
  • Database - wpdb usage, custom tables, options API
  • REST API - Endpoint registration, schema validation, permissions
  • Block Development - block.json, render.php, editor patterns
skill: wp-development-patterns

wp-ai-building-blocks

WordPress AI

Building AI-powered features for WordPress? This skill covers the WordPress AI Building Blocks ecosystem--the Abilities API for registering capabilities, the AI Client SDK for provider-agnostic AI generation, and the MCP Adapter for connecting AI tools. Includes 4 reference files with patterns and examples.

Reference Files

  • Abilities API patterns and registration
  • AI Client SDK usage and provider configuration
  • MCP Adapter setup and tool registration
  • Integration patterns and best practices
skill: wp-ai-building-blocks

setup

Setup

Configure the compound-wordpress-engineering plugin for your workflow. Choose agent presets (Comprehensive, Lean, WordPress-focused) and focus areas (Security, Performance, Traceability) to customize which agents run during reviews and how the plugin behaves.

Configuration Options

  • Comprehensive preset - All agents enabled
  • Lean preset - Core agents only
  • WordPress preset - WordPress-specific agents prioritized
  • Traceability focus - Call-chain-verifier and related agents
skill: setup

Testing & Quality (5)

Testing infrastructure, static analysis, and WordPress Playground for disposable test environments. These skills make the test-and-review loop self-contained.

wp-testing

Testing

Scaffold and run WordPress test suites--PHPUnit for unit tests, WP_UnitTestCase for integration tests, and Playwright for browser testing. Includes scripts for scaffolding test infrastructure and running suites, plus reference docs for PHPUnit setup, test patterns, TDD workflow, and fixture patterns.

Key Features

  • Scaffolding - Auto-generates phpunit.xml.dist, bootstrap.php, and sample tests for plugins, themes, or blocks
  • Test Pyramid - Unit, integration, REST API, WP-CLI, and browser test guidance
  • TDD Workflow - Red-green-refactor adapted for WordPress development
  • Fixture Patterns - Factory methods, setUp/tearDown, JSON fixtures
skill: wp-testing

wp-playground

Testing

Start disposable WordPress instances for testing using @wp-playground/cli. Auto-detects plugin/theme/block projects and mounts them correctly. Includes scripts for start/stop/status, blueprint patterns for configuring test environments, and an evaluation procedure for comparing Playground implementations.

Key Features

  • Auto-detection - Identifies plugin, theme, or block projects and mounts accordingly
  • Blueprint support - JSON blueprints for reproducible test environments
  • Quick access - Default credentials admin/password at localhost:9400
  • Integration - Works with PHPUnit, wp-browser, and Playwright for automated testing
skill: wp-playground

wp-phpcs

Linting

Run PHP_CodeSniffer with WordPress Coding Standards on changed files. Includes setup and run scripts, plus phpcs.xml.dist configuration templates for plugins and themes. Integrates with the wp-php-reviewer agent for automated review workflows.

Key Features

  • Setup automation - Installs PHPCS + WPCS via Composer
  • Changed-files mode - Runs only on files changed since the base branch
  • Auto-fix - phpcbf integration for automatic style corrections
  • JSON output - Structured results for agent consumption
skill: wp-phpcs

wp-phpstan

Analysis

Run PHPStan static analysis with the szepeviktor/phpstan-wordpress extension for WordPress-aware type checking. Catches undefined methods, incorrect function signatures, and type errors that PHPCS misses. Includes setup scripts, run scripts, and phpstan.neon configuration templates.

Key Features

  • WordPress-aware - Understands WordPress function signatures and global variables
  • Configurable levels - Analysis levels 0-9 for gradual adoption
  • Baseline management - Ignore existing errors, catch new ones
  • Changed-files mode - Analyze only files modified since the base branch
skill: wp-phpstan

wp-eslint

Linting

Run ESLint with @wordpress/eslint-plugin for JavaScript and TypeScript code quality. Covers block editor patterns, Interactivity API stores, and @wordpress/* package usage. Includes run scripts and .eslintrc.json configuration templates.

Key Features

  • WordPress rules - @wordpress/eslint-plugin for WordPress JS conventions
  • Auto-fix - ESLint --fix for automatic corrections
  • Changed-files mode - Lint only JS/TS files changed since the base branch
  • JSON output - Structured results for agent consumption
skill: wp-eslint

These skills teach Claude specific coding styles, architectural patterns, and development approaches. Use them when you want code that follows a particular philosophy.

agent-native-architecture

AI

Build AI agents using prompt-native architecture where features are defined in prompts, not code. When creating autonomous agents, designing MCP servers, or implementing self-modifying systems, this skill guides the "trust the agent's intelligence" philosophy.

Key Patterns

  • Prompt-Native Features - Define features in prompts, not code
  • MCP Tool Design - Build tools agents can use effectively
  • System Prompts - Write instructions that guide agent behavior
  • Self-Modification - Allow agents to improve their own prompts
skill: agent-native-architecture

frontend-design

Design

You've seen what AI usually generates: Inter font, purple gradients, rounded corners on everything. This skill teaches Claude to design interfaces that don't look like every other AI-generated site. It's about purposeful typography, unexpected color palettes, and interfaces with personality.

Focus Areas

  • Typography with distinctive font choices
  • Color & theme coherence with CSS variables
  • Motion and animation patterns
  • Spatial composition with asymmetry

skill: frontend-design

create-agent-skills

Meta

The master template for building skills. Covers skill architecture, the router pattern for complex multi-step skills, progressive disclosure design, SKILL.md structure, and asset management. When you need to build a skill, start here.

Workflows Included

  • create-new-skill - Start from scratch
  • add-reference - Add reference documentation
  • add-template - Add code templates
  • add-workflow - Add step-by-step procedures
  • audit-skill - Validate skill structure

skill: create-agent-skills

skill-creator

Meta

The simpler, step-by-step version of create-agent-skills. When you just want a checklist to follow from blank file to packaged skill, use this. Less theory, more "do step 1, then step 2."

  2. Plan reusable skill contents
  3. Initialize skill using template
  4. Edit skill with clear instructions
  5. Package skill into distributable format
  6. Iterate based on testing feedback
skill: skill-creator

Ideation

Structured brainstorming and ideation workflows. When you need to explore possibilities systematically rather than just throwing ideas at a wall, this skill provides frameworks for divergent thinking, idea evaluation, and convergence.

compound-docs

Docs

You just fixed a weird build error after an hour of debugging. Tomorrow you'll forget how you fixed it. This skill automatically detects when you solve something (phrases like "that worked" or "it's fixed") and documents it with YAML frontmatter so you can find it again. Each documented solution compounds your team's knowledge.

Auto-Triggers

Phrases: "that worked", "it's fixed", "working now", "problem solved"

Categories

  • build-errors/
  • test-failures/
  • runtime-errors/
  • performance-issues/
  • database-issues/
  • security-issues/
skill: compound-docs

document-review

Review

Review and improve documentation quality. Checks for clarity, completeness, accuracy, and consistency across your docs. Useful for README files, API documentation, and user guides. 

skill: document-review

file-todos

Workflow

Your todo list is a bunch of markdown files in a todos/ directory. Each filename encodes status, priority, and description. No database, no UI, just files with YAML frontmatter. When you need to track work without setting up project management software, this is the system.

File Format

# Naming convention
{issue_id}-{status}-{priority}-{description}.md

# Examples
001-pending-p1-security-vulnerability.md
002-ready-p2-performance-optimization.md
003-complete-p3-code-cleanup.md
skill: file-todos

Integration

  • Works with /work for parallel feature development
skill: git-worktree

skill: orchestrating-swarms

Image Generation (1)

Generate images with AI. Not stock photos--images you describe and the model creates.

gemini-imagegen

AI Images

Need a logo with specific text? A product mockup on a marble surface? An illustration in a kawaii style? This skill wraps Google's Gemini image generation API. You describe what you want, it generates it. You can edit existing images, refine over multiple turns, or compose from reference images. All through simple Python scripts.

Features

Text-to-image generation
Image editing & manipulation
Multi-turn iterative refinement
Multiple reference images (up to 14)
Google Search grounding (Pro)

Available Models

Model Resolution Best For
gemini-2.5-flash-image 1024px Speed, high-volume tasks
gemini-3-pro-image-preview Up to 4K Professional assets, complex instructions

Quick Start

# Text-to-image
python scripts/generate_image.py "A cat wearing a wizard hat" output.png

# Edit existing image
python scripts/edit_image.py input.png "Add a rainbow in the background" output.png

# Multi-turn chat
python scripts/multi_turn_chat.py

Requirements

GEMINI_API_KEY Required environment variable
google-genai Python package
pillow Python package for image handling 
skill: gemini-imagegen
